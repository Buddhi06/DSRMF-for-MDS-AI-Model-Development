TM Phase 1: Objective Identification

Objective identification is the first phase of the TM process. In this phas, the security properties that the MDS AI model should have need to be identified and defined.

To develop the threat and vulnerability landscape this framework considers the following six security properties:

Property AI-specific Definition
Confidentiality When a model is used to perform an inference, only the input and output of the model are revealed.
Integrity Information used or generated throughout the AI model’s life-cycle cannot be changed or added to by unauthorised third parties.
Availability When inputs are presented to the AI model, it produces meaningful results that are easily distinguished from random noise.
Authenticity The output value that the AI model produces has been proven to be produced by the model.
Authorisation The AI model can only accept inputs from authorised parties and return the relevant outputs.
Non-repudiation A model's output cannot be denied to have been produced by it.
